Fun with JSF

Published on Mon 02 September 2013 under

I tried answering a StackOverflow question marginally related to my current interest (Java EE). It took me 4 hours and here's what I learned:

  • Any validation error in the Process Validations phase of the JSF lifecycle prevents the entire Update Model Values phase.

  • Even a <h:selectOneMenu/> may throw validation errors; this happens when the element class has a buggy equals() method. If the method returns false negatives the selected element may not be found in the list.

    • Such an error apparently doesn't show up in the message tag.

  • There exists a source jar for the version of jsf-impl contained in JBoss 7.1.1 but it's not available in any Maven repository (neither the central one nor JBoss' own).

  • When matching Maven artifacts to provided server-side libraries, utmost care must be taken to get the version strings right...
  • Eclipse doesn't fetch source files for a Maven-managed JAR immediately, some clicking around in the editor view is required.
  • To turn on client-side RichFaces logging into the browser console, insert <script>RichFaces.setLog(console)</script> into a template.
